## Goal
Access and edit `photosync` Laravel project from VS Code on local machine using Remote-SSH.

---

## Actions & Commands

### System Setup
- Install Google Cloud SDK:
  ```bash
  sudo apt update && sudo apt install -y curl apt-transport-https ca-certificates gnupg
  curl -fsSL https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo gpg --dearmor -o /usr/share/keyrings/cloud.google.gpg
  echo "deb [signed-by=/usr/share/keyrings/cloud.google.gpg] https://packages.cloud.google.com/apt cloud-sdk main" | sudo tee /etc/apt/sources.list.d/google-cloud-sdk.list
  sudo apt update && sudo apt install -y google-cloud-sdk
  ```

### Authentication & Project Configuration
- Login and set project: `gcloud auth login && gcloud config set project photosync-backend-479418`

### SSH Configuration
- Fix key permissions: `$ chmod 600 ~/.ssh/google_compute_engine && chmod 644 ~/.ssh/google_compute_engine.pub`

- Create SSH config: `$ nano ~/.ssh/config`
  
  Add configuration:
  ```
  Host google-cloud-shell
    HostName 34.75.246.76
    User etters_ayoub
    Port 6000
    IdentityFile ~/.ssh/google_compute_engine
    StrictHostKeyChecking no
  ```

### Connection Testing
- Test Cloud Shell: `$ gcloud cloud-shell ssh`
- Test SSH config: `$ ssh google-cloud-shell`

### VS Code Integration
- Install **Remote - SSH** extension
- Open Command Palette: `Ctrl + Shift + P`
- Select: **Remote-SSH: Connect to Host...**
- Choose: `google-cloud-shell`
- Open `photosync` folder in remote workspace

## Expected Outcomes
- Successful VS Code connection to Google Cloud Shell
- Full remote development environment functionality
- Direct access to `photosync` Laravel project
